# 🔣 알고리즘 : Dynamic Programming

동적계획법(Dynamic Programming)의 구현 방식에는 두가지가 있다.
1. Top-Down 방식의 **재귀(recursion)** 마지막에서 처음으로
2. Bottom-Up 방식의 **반복(iteration)** 처음에서 마지막으로

두 가지 방식에서 **반복되는 문제는 한 번만 푼다** 를 추가하면 Dynamic Programming 의 요구 조건을 만족시킨다.<br>
반복되는 문제를 한 번만 풀기 위해서는 Memoization(배열, 리스트, 집합 등을 사용하여)을 구현 하여야 한다.


## ❗ 알고리즘

LCS(Longest Common Subsequence) 문제와 매우 유사한 문제  
  
문자열의 부분집합을 어떻게 바라봐야 하는지 알 수 있는 문제  
  
문자열을 행렬로 구성하면 문자가 추가될 때 행할 동작을 계산 가능하다

가령 x,y 번 째 문자열 집합을 x -> y 로 바꾸는데 드는 비용이라고 하면

x -> y 로 바꾸는 법은

x -> y-1 로 바꾸고 y 번째 char 를 추가하는 법 ... 1

x 번째 char 빼고 x-1 -> y 로 바꾸는 법 ... 2

x-1 -> y-1 로 바꾸고 y 번 째 char 를 따로 연산해주는 것 (교체의 동작을 의미) ... 3

세가지 가능한 경우의 최솟값을 x -> y 에 채워주면 됨
