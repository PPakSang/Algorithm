# 🔣 알고리즘 : Greedy

탐욕스럽고 욕심많은 전략을 택한다 해서 Greedy 하다고 불리는 이 알고리즘은<br>
나에게 당장 가장 많은 이득을 가져다 줄 것으로 예상되는 행동을 취하는 방식이다. ~~팃포텟?~~<br>
알고리즘을 학습하면서 느낀점은 수학적인 문항을 풀 때 자주나오고, 그렇기 때문에 조건식을 잘 만들어야한다.

## ❗ 알고리즘

덧셈과 뺄셈이 교차하는 다항식(polynomial) 에서 어떻게 괄호를 쳤을 때 전체 값이 최소가 될 수 있는지<br>
수학적으로 생각할 수 있는 문제. 덧셈을 먼저 계산한 뒤 뺄셈을 수행하면 그 값이 최소가 된다.<br><br>

Stack 두개를 만들어 하나는 숫자를 저장, 다른 하나는 연산자를 저장했다.<br>
모든 숫자 및 연산자를 스택에 담은 뒤 하나씩 뽑는데, +가 나오면 temp 에다가 더해주고 - 가 나오면 temp 를 전체에 빼고 temp 를 0으로 만들었다.<br><br>

StringTokenizer 말고도 split 으로도 풀어볼 수 있는 문제

## ❗ 알고리즘 외

StringTokenizer(str, delim, returndeli) 의 형태로 delimeter 추가해서 사용가능<br>
split 은 리턴값이 배열로 나온다. 정규식으로 파싱하기 때문에 +는 \\+ 로 해야함(meta character 라서)<br>
