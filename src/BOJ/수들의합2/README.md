## ❗ 알고리즘
Indexing 할 수 있는 객체가 주어졌을 때 한번 생각해 볼법한 방식<br>
마치 n개의 손가락으로 배열의 이것저것을 한번 선택해보듯<br><br>
이 문제에서는 두 개의 손가락으로 배열의 부분을 선택해서 손가락 사이에 있는 숫자를 모두 더한 결과의 출력이 필요했음
<br><br>
배열을 생성해두고 두개의 포인터 i, j 를 이용해 2차원 배열을 탐색하듯 구현하였다.<br>
Stream 으로 깔끔하게 짤 수 있는 방법이 없을까 고민하였지만 ~~조금 더 고민해봐야겠다...~~

## ❗ 알고리즘 외

JAVA 언어로 사용자 입력(System.in) 받아오는 대표적인 방식으로
1. new Scanner(System.in)
2. new BufferedReader(new InputStreamReader(System.in))

Scanner 는 내부적으로 입력 구문을 분석하기 때문에 다소 속도가 느림<br>
BufferedReader 는 Stream 을 단순히 읽어서 저장하기 때문에 개발자가 문자 파싱을 해줘야함
