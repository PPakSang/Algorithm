## ☑️풀이

stages range 를 확인해보니 200,000 -> stage 는 2번이상 살펴보면 안되겠다고 판단
  
stage 값의 범위는 500 으로 stage value(500) * stages(200,000) 이런식으로 구성하는 것도 너무 느리다고 생각
  
젤 좋은건 200,000 한번 확인하고 stage 값들에 대해서 500 * 500 싸이클이 추가되는 것이 가장 최적일 것이라고 판단  

각 스테이지 별로 사람 수를 저장하고(200,000 명을 500개의 stage 에 mapping) 이 배열을 사용  
  
첫 stage 에 도달한 사람은 이후 stage 에 도달한 합과 같고 DP 가 생각이 났다    
  
따라서 각 stage 별 실패자 수와 도달한 사람 수를 구할 수 있기 때문에 답을 구할 수 있을거라 생각했는데...  
  
각 rate 정렬 순으로 index 를 어떻게 저장할까 마지막 고민이 필요했다.  
  
둘을 묶어서 같이 움직인다면 rate 를 정렬하면 index 도 정렬한 셈이기 때문에 하나의 class 로 묶어줬음
