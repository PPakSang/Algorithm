# 🔣 숫자문자열과영단어

정리할게 너무나도 많은 문제, 정말 갈길이 먼 것 같다.<br><br>

처음 Node 객체를 만들어서 연결 삭제를 구현해야한다고 생각했다.<br>
구현을 하던 중 자를 때 마다 노드를 처음부터 탐색해야하는 딱 봐도 비효율적인 코드가 만들어지고 있었고<br>
어떤 방법이 좋을까 고민하다가 도저히 모르겠어서 1fe 한테 물어봤다.<br><br>

이 문제의 핵심은 지워진 행 기록하고, 마지막에 모든 행의 상태를 만들어내는 것이다.<br>
즉 지울 행을 기록으로 남기고, 이전 행을 되돌리기 위해선 기록으로 남긴 행을 되돌리기만 하면 된다.<br>
현재 포인터 기준으로 데이터가 위로 쌓이면 포인터를 바꿀 필요없고, 밑에 쌓이면 포인터의 인덱스를 하나 올려줘야한다.<br><br>

마무리도 꽤나 중요한데 동일한 인덱스를 삭제한 기록이 연속으로 들어오면 가장 최근에 들어온 기록이 더 큰 인덱스 일 것이므로<br>
StringBuilder 의 insert 를 사용하여 처리를 해주어야 한다.

☑️StringBuilder.append() StringBuilder.insert()

append 횟수 가 StringBuilder 의 index 를 결정하고, insert 를 하면 해당 위치에 있는 문자열은<br>
**우측** 으로 밀어내고 새로운 문자열을 추가한다.
